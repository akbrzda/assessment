## КОНТЕКСТ: ЧТО УЖЕ СДЕЛАНО, ЧТО НЕ СООТВЕТСТВУЕТ

### Реализовано и работает

- Таблицы `courses`, `course_sections`, `course_topics`, `course_user_progress`, `course_section_user_progress`, `course_topic_user_progress`, `course_user_snapshots`
- Admin API: CRUD курсов, разделов и тем, публикация, архивация, аудит
- User API: список курсов, карточка курса (с разделами + темами), старт, прогресс, завершение раздела, итоговая аттестация
- User frontend: каталог курсов (вкладка в AssessmentsView), детальная страница курса (показываются разделы, темы — в работе)
- Admin frontend: список курсов (CoursesView), редактор курса (CourseEditorView — обновление в работе)
- Feature-flag `COURSES_ENABLED`, версионирование, snapshot при старте
- Валидация публикации: `finalAssessmentId` — опциональна

### Оставшиеся расхождения с docs/courses.md

1. **User API тем** — нет endpoint фиксации просмотра материала, endpoint завершения модуля не переименован в section
2. **User frontend** — темы не отображаются внутри разделов, кнопки «Изучить материал» / «Пройти тест темы» отсутствуют
3. **Назначение курсов** — нет привязки по должностям и филиалам; нет ручного назначения; все опубликованные курсы видны всем
4. **Просмотр прогресса пользователей** — нет admin API и UI для списка зачисленных пользователей и их прогресса
5. **Сброс прогресса** — не реализован ни API, ни UI
6. **Аналитика** — события курсов не отправляются
