- [ ] Переиспользовать текущую логику модуля аттестаций без отдельного дублирования
- [ ] Для теста темы добавить настройку количества попыток или режим без ограничений
- [ ] Пользовательский сценарий прохождения теста темы должен быть аналогичен аттестации, а результат показывается через модальное окно без отдельной страницы результатов

## 5. Итоговая аттестация курса — P0

- [ ] Сделать итоговую аттестацию обязательной для каждого курса
- [ ] Создание и редактирование итоговой аттестации перенести внутрь шага 3 мастера курса
- [ ] Добавить настройки порога прохождения, лимита времени и количества попыток или безлимитного режима
- [ ] Завершение курса фиксировать только после успешной итоговой аттестации

## 6. Назначение и сроки действия курса — P1

- [ ] Поддержать бессрочный режим курса
- [ ] Поддержать режим «N дней от даты назначения пользователю»
- [ ] Поддержать режим фиксированных дат действия курса
- [ ] Корректно сохранять дату назначения и индивидуальный дедлайн пользователя
- [ ] Добавить сценарий закрытия курса администратором с пользовательским статусом «Закрыт»

## 7. Пользовательский сценарий прохождения — P1

- [ ] На странице обучения сделать вкладку «Курсы» активной по умолчанию
- [ ] В карточке курса показать срок действия, количество тем, количество тестов, прогресс и статус
- [ ] На странице курса показывать доступные и заблокированные темы с понятной индикацией
- [ ] После завершения всех обязательных тем автоматически открывать итоговую аттестацию
- [ ] Необязательные темы не должны влиять на общий прогресс и завершение курса

## 8. Аналитика и статистика курса — P1

- [ ] Показать для курса количество пользователей: назначено, начали, проходят, завершили
- [ ] Добавить учет общего времени, проведенного в курсе
- [ ] Добавить количество попыток по тестам тем и итоговой аттестации
- [ ] Добавить средний балл по курсу и по итоговой аттестации
- [ ] Подготовить удобный отчет по прогрессу пользователей внутри курса

## 9. Техническая стабилизация и регрессия — P1

- [ ] Покрыть новый сценарий тестами backend и frontend
- [ ] Проверить обратную совместимость текущего прогресса пользователей
- [ ] Обновить API-контракты и документацию после внедрения изменений
- [ ] Провести регрессионную проверку курсов, теории и аттестаций перед релизом
