# Заключение по AUDIT_REPORT

Источник: `docs/AUDIT_REPORT.md` (дата отчета: 2025-02-14)  
Актуализация: 2026-04-11

## Общий вывод

`AUDIT_REPORT.md` в целом корректен по сути и остается актуальным по большинству ключевых рисков.  
Часть проблем подтверждена без изменений, часть — частично закрыта, но не до production-уровня.

## Статус по ключевым пунктам аудита

## Критичные

1. Фолбэк-секреты JWT — **Не закрыто**.
- fallback присутствуют в конфиге и auth-слое.

2. CORS + credentials с открытым origin — **Не закрыто**.
- при пустом `ALLOWED_ORIGINS` остается `origin: true`.

3. CSRF-защита refresh cookie-flow — **Не закрыто**.
- явный CSRF-механизм не реализован.

## Высокие

4. `localStorage` для access-token в админке — **Не закрыто**.

5. Защита от brute force логина — **Не закрыто**.

6. Проверка свежести Telegram `auth_date` — **Не закрыто**.

7. Ограничение менеджера по филиалу в `canEditUser` — **Частично закрыто/неполно**.
- часть ограничений есть в контроллерах, но middleware-проверка филиала не доведена.

## Средние

8. Кеш без user-scope — **Частично закрыто**.
- кеширование есть, но не везде учитывается пользовательский контекст.

9. Отсутствие пагинации в крупных выборках — **Не закрыто**.

10. Расхождения модулей доступа между backend/frontend — **Частично закрыто**.
- логика есть, но перечни модулей по умолчанию не везде согласованы.

11. Security headers (`helmet`, CSP и т.д.) — **Не закрыто**.

12. Cloud storage fallback (DELETE) — **Не закрыто**.

13. Аудит фактически не сохраняется — **Не закрыто**.
- вызовы есть, но сохранение/доставка не реализованы как полноценный pipeline.

## Низкие

14. `SELECT *` — **Не закрыто** (встречается в ряде мест).

15. Избыточные `console.log` — **Не закрыто** (особенно в debug-потоках).

## Дополнительно выявлено при актуализации

1. Ошибка в SQL для напоминаний по незавершенным попыткам (`attempts` вместо корректной таблицы).
2. Дефект ветвления в логике проверки ответов `matching`.
3. Неполноценная реализация фиксированного порядка вопросов/вариантов в попытке при возобновлении.
4. Проблемы дисциплины миграций:
   - дублирующиеся номера миграций (`002_*`),
   - отсутствие истории применения миграций.

## Что считать закрытием аудита

Аудит можно считать закрытым только после выполнения всего P0 и ключевых пунктов P1:

1. Безопасный auth baseline (JWT/CORS/CSRF/rate-limit).
2. Реальный аудит действий (persist + доставка в Telegram thread).
3. Полная изоляция менеджеров по филиалам.
4. Исправленные дефекты аттестаций (matching/SQL/возобновление попытки).
5. Базовые quality gates (lint/test/build) в CI до deploy.

## Финальное заключение

Аудит 2025 года остается релевантным и должен рассматриваться как действующий.  
Текущее состояние проекта требует обязательной фазы стабилизации и безопасности перед активной реализацией крупных продуктовых фич.
