# БИЗНЕС-ТРЕБОВАНИЯ: МОДУЛЬ «КУРСЫ»

> Актуализировано 22.04.2026 по согласованным правилам.
> Этот документ является основным источником терминологии и пользовательского сценария.

---

## 1. Обязательная терминология

### 1.1 Единая бизнес-модель

Во всех интерфейсах, документации и постановках используется только следующая терминология:

- **Курс** — целиком вся обучающая программа
- **Тема курса** — крупный логический блок внутри курса
- **Подтема** — часть темы курса, аналог параграфа или шага чтения

Аналогия:

- Курс = книга
- Тема курса = глава
- Подтема = параграф

### 1.2 Что запрещено использовать в бизнес-описании

Для пользователя и заказчика больше не используем как основные термины:

- «модуль»
- «раздел»

Если такие слова временно остаются во внутреннем коде или БД, это считается техническим наследием, а не бизнес-термином.

---

## 2. Назначение модуля

Модуль «Курсы» нужен для последовательного обучения сотрудников:

1. сотрудник изучает материал по подтемам;
2. при необходимости проходит тест темы;
3. после завершения обязательных тем открывается итоговая аттестация;
4. успешная итоговая аттестация завершает курс.

---

## 3. Структура курса

### 3.1 Иерархия

```text
Курс
+- Тема курса 1
|  +- Подтема 1
|  +- Подтема 2
|  +- Тест темы (опционально)
+- Тема курса 2
|  +- Подтема 1
|  +- Подтема 2
|  +- Тест темы (опционально)
+- Итоговая аттестация (обязательно)
```

### 3.2 Состав темы курса

Каждая тема курса содержит:

- название;
- признак обязательности;
- список подтем;
- опциональный тест темы.

### 3.3 Состав подтемы

Каждая подтема содержит:

- название;
- теоретический контент;
- порядок внутри темы;
- расчетное время чтения.

### 3.4 Правила обязательности

- **обязательные темы курса** влияют на прогресс и открытие итоговой аттестации;
- **необязательные темы курса** доступны для прохождения, но не влияют на общий процент завершения курса.

---

## 4. Административный сценарий

### 4.1 Общий сценарий для админа

Администратор переходит на страницу курсов и может:

- видеть список всех курсов;
- фильтровать курсы;
- создавать новый курс;
- редактировать уже созданный курс;
- смотреть статистику и прогресс пользователей.

### 4.2 Создание курса — пошаговый мастер по аналогии с формой создания аттестации

#### Шаг 1. Основная информация

Администратор заполняет:

- название курса;
- описание курса;
- назначения по должностям;
- назначения по филиалам;
- при необходимости ручные назначения пользователям;
- режим срока действия.

#### Шаг 2. Темы курса и подтемы

Администратор:

- создает тему курса;
- задает ее название;
- отмечает, обязательная она или нет;
- добавляет подтемы;
- настраивает порядок прохождения;
- при необходимости включает тест темы.

#### Шаг 3. Итоговая аттестация

Администратор:

- обязательно включает итоговую аттестацию;
- создает ее прямо внутри курса;
- задает порог прохождения;
- задает ограничение по времени;
- задает количество попыток или безлимитный режим.

#### Шаг 4. Предварительный просмотр

Администратор видит:

- карточку курса;
- структуру тем и подтем;
- параметры прохождения;
- итоговую аттестацию;
- итоговый вид перед публикацией.

Логика перехода между шагами, визуальный прогресс и навигация «Назад / Далее / Публикация» должны быть такими же, как в текущей форме создания аттестации.

После проверки курс можно опубликовать.

---

## 5. Правила создания тестов внутри курса

### 5.1 Тест темы

Тест темы создается прямо внутри курса и использует текущую логику модуля аттестаций как основу.

Сценарий прохождения теста темы для пользователя должен быть максимально близок к обычной аттестации по шагам и поведению процесса.

Настраиваются:

- вопросы;
- варианты ответов;
- порог прохождения;
- лимит времени;
- количество попыток или бесконечные попытки.

### 5.2 Итоговая аттестация

Итоговая аттестация:

- является **обязательной** для каждого курса;
- создается в рамках курса, а не отдельно от него;
- определяет, считается курс завершенным или нет.

---

## 6. Срок действия курса

### 6.1 Поддерживаемые режимы

Для курса должны поддерживаться три режима действия:

1. **Бессрочный курс**
   - курс не ограничен по времени.

2. **Срок от даты назначения**
   - например, 90 дней от момента назначения конкретному пользователю.

3. **Фиксированные даты**
   - задается общее окно действия, например с 01.05 по 31.07.

### 6.2 Бизнес-правило по закрытию

Статус **«Закрыт»** означает, что курс был закрыт со стороны администратора.

Это отдельный бизнес-статус и его нельзя смешивать:

- с состоянием «не начат»;
- с неуспешным прохождением;
- с окончанием срока действия.

---

## 7. Пользовательский сценарий

### 7.1 Точка входа

На странице обучения у пользователя есть переключение между курсами и аттестациями.

**По умолчанию активна вкладка «Курсы».**

### 7.2 Карточка курса

Пользователь в списке курсов видит:

- название курса;
- срок действия;
- количество тем курса;
- количество тестов внутри курса;
- процент прогресса;
- текущий статус.

### 7.3 Статусы курса для пользователя

| Статус     | Значение                                                |
| ---------- | ------------------------------------------------------- |
| Не начат   | пользователь еще не начинал курс                        |
| В процессе | пользователь проходит курс                              |
| Завершен   | обязательные темы завершены и итоговая аттестация сдана |
| Закрыт     | курс закрыт администратором                             |

### 7.4 Внутри курса

На странице курса пользователь видит:

- ту же сводную информацию по курсу;
- список тем курса;
- статус каждой темы;
- замки на недоступных темах.

Если курс только начат:

- первая доступная обязательная тема открыта;
- остальные темы закрыты до выполнения условий последовательности.

### 7.5 Страница темы курса

Страница темы выглядит по аналогии со страницей теории:

- сверху показывается прогресс по подтемам;
- внутри темы есть последовательный переход по подтемам;
- следующая подтема открывается только после завершения текущей;
- если для текущей подтемы не истекло расчетное время чтения, перейти дальше нельзя.

### 7.6 Автозавершение по времени чтения

Для каждой подтемы и темы время чтения определяется по аналогии с теорией — на основе количества слов.

После истечения обязательного времени:

- подтема считается прочитанной;
- пользователь может перейти дальше;
- прогресс темы обновляется автоматически.

### 7.7 Тест темы

Когда все подтемы темы завершены:

- пользователь может начать тест темы;
- прохождение должно идти по тому же пользовательскому сценарию, что и обычная аттестация;
- итог результата показывается через модальное окно, без отдельной страницы результатов.

### 7.8 Итоговая аттестация

После завершения всех **обязательных** тем курса:

- открывается итоговая аттестация;
- она работает почти так же, как текущая аттестация;
- успешное прохождение означает полное завершение курса.

---

## 8. Правила прогресса

### 8.1 Что влияет на общий прогресс

В общий прогресс курса входят только обязательные темы курса.

Формула:

```text
Прогресс курса = завершенные обязательные темы / общее число обязательных тем x 100%
```

### 8.2 Что не влияет на общий прогресс

Не влияют:

- необязательные темы;
- незавершенные необязательные подтемы;
- просмотр дополнительных материалов вне обязательного маршрута.

### 8.3 Условие завершения темы курса

Тема считается завершенной, когда:

1. завершены все подтемы внутри темы в актуальном порядке;
2. если включен тест темы — он сдан успешно.

### 8.4 Условие завершения курса

Курс считается завершенным, когда:

1. завершены все обязательные темы курса;
2. итоговая аттестация сдана успешно.

---

## 9. Попытки и ограничения

### 9.1 Для тестов темы и итоговой аттестации

Должны поддерживаться оба режима:

- ограниченное количество попыток;
- бесконечное количество попыток.

### 9.2 Поведение при провале

Если попытки еще доступны:

- пользователь может пройти тест повторно.

Если попытки закончились:

- тест блокируется до решения администратора или изменения настроек курса.

---

## 10. Состояния подтемы в интерфейсе

### 10.1 Возможные состояния

Каждая подтема в списке отображается в одном из следующих состояний:

| Состояние               | Визуал                 | Описание                                                                    |
| ----------------------- | ---------------------- | --------------------------------------------------------------------------- |
| Не начата               | без галочки, без замка | подтема доступна, но ещё не пройдена                                        |
| В процессе              | прогресс % без галочки | пользователь начал, но не завершил                                          |
| Пройдена                | зелёная галочка        | подтема завершена                                                           |
| Заблокирована           | замок + «Закрыто»      | недоступна до прохождения предыдущей                                        |
| Пройдена, заблокирована | галочка + замок        | была пройдена ранее, но заблокирована из-за вставки новой подтемы перед ней |

### 10.2 Галочка и замок одновременно

Состояние «пройдена + заблокирована» возникает, когда:

1. пользователь прошёл подтему N;
2. администратор вставил новую подтему перед N (или между ранее пройденными);
3. система обнаружила нарушение последовательности — подтема N заблокирована до прохождения новой.

Галочка сохраняется как исторический факт прохождения и не убирается.
Замок отражает текущую недоступность из-за нарушенной последовательности.

### 10.3 Автоматическое восстановление после прохождения новой подтемы

Когда пользователь проходит вставленную подтему:

- все последующие подтемы, которые были пройдены ранее, автоматически считаются завершёнными;
- их статус восстанавливается до «пройдена» без повторного прохождения;
- замки с них снимаются автоматически;
- если все подтемы темы снова в статусе «пройдена» — тест темы разблокируется.

Пример:

```text
До вставки:
Подтема 1 ✅ → Подтема 2 ✅ → Подтема 4 ✅

После вставки Подтемы 3:
Подтема 1 ✅ → Подтема 2 ✅ → [NEW] Подтема 3 ▶️ → Подтема 4 ✅🔒

После прохождения Подтемы 3:
Подтема 1 ✅ → Подтема 2 ✅ → Подтема 3 ✅ → Подтема 4 ✅ → Тест темы ▶️
```

### 10.4 Требование к хранению прогресса подтем

В базе данных для каждого пользователя по каждой теме хранится не только флаг `is_completed`, но и **порядковый номер последней пройденной подтемы** (`last_completed_order`).

Это необходимо для корректного пересчёта статусов при добавлении или перестановке подтем администратором.

## 11. Аналитика и статистика

Администратор должен видеть по каждому курсу:

- количество пользователей, которым назначен курс;
- количество пользователей, которые начали курс;
- количество пользователей, которые проходят курс сейчас;
- количество пользователей, которые завершили курс;
- средний балл;
- общее время прохождения курса;
- количество попыток по тестам тем;
- количество попыток по итоговой аттестации.

Дополнительно желательно:

- фильтрация по филиалу, должности, периоду;
- проваленные темы и узкие места;
- список пользователей с детальным прогрессом.

---

## 12. Ключевые ограничения

1. Курс не публикуется без тем курса.
2. Тема курса не создается без хотя бы одной подтемы.
3. Итоговая аттестация обязательна.
4. Тесты темы и итоговой аттестации создаются внутри курса.
5. Необязательные темы не должны ломать завершение курса.
6. Пользовательская вкладка курсов должна быть основной точкой входа в обучение.
7. Галочка на подтеме не убирается никогда — она отражает исторический факт прохождения.
8. При вставке новой подтемы в уже опубликованный курс система обязана пересчитать статусы доступности для всех пользователей, у которых есть прогресс по данной теме.

## 13. Roadmap реализации

### Этап 1. Унификация терминологии

- обновить тексты UI и документацию;
- закрепить модель «Курс -> Тема курса -> Подтема».

### Этап 2. Новый wizard для админа

- основная информация;
- темы и подтемы;
- итоговая аттестация;
- предпросмотр и публикация.

### Этап 3. Пользовательское прохождение

- вкладка «Курсы» по умолчанию;
- последовательные подтемы;
- таймер чтения и автозавершение;
- страничное прохождение теста темы + показ результата в popup/модальном окне.

### Этап 4. Сроки и статусы

- бессрочный режим;
- срок от назначения;
- фиксированные даты;
- статус «Закрыт» со стороны админа.

### Этап 5. Аналитика

- время в курсе;
- попытки;
- средний балл;
- статистика начавших, проходящих и завершивших.
