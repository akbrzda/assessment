## 1. ВВЕДЕНИЕ

### 1.1 Назначение системы

Модуль "Курсы" предназначен для организации структурированного обучения сотрудников с последовательным освоением материала и контролем знаний через аттестации.

### 1.2 Основные цели

1. **Обучение персонала** — предоставление теоретических материалов по темам
2. **Контроль знаний** — проверка освоения материала через тесты
3. **Последовательное прохождение** — блокировка следующих тем до завершения предыдущих
4. **Переаттестация** — возможность периодической проверки знаний без повторного прохождения курса

### 1.3 Целевая аудитория

- Сотрудники ресторанов (повара, официанты, администраторы)
- Менеджеры филиалов
- Учебные координаторы
- Администраторы системы

## 2. ОСНОВНЫЕ СУЩНОСТИ

### 2.1 Курс (Course)

**Определение:** Курс — это структурированная программа обучения, состоящая из разделов, тем и аттестаций.

**Характеристики:**

- Название курса
- Описание курса
- Статус (черновик / опубликован / архивирован)
- Целевая аудитория (должности и филиалы)
- Итоговая аттестация (опционально)

**Примеры:**

- "Обучение поваров"
- "Стандарты обслуживания гостей"
- "Работа с кассовым оборудованием"

### 2.2 Раздел (Section)

**Определение:** Раздел — это тематическая группа внутри курса, объединяющая несколько связанных тем.

**Характеристики:**

- Название раздела
- Описание раздела
- Порядковый номер в курсе
- Обязательность (обязательный / необязательный)
- Аттестация раздела (тест, который сдаётся после изучения всех тем раздела)

**Примеры разделов для курса "Обучение поваров":**

- Раздел 1: "Пицца"
- Раздел 2: "Закуски"
- Раздел 3: "Заготовка ингредиентов"
- Раздел 4: "Закрытие смены"

### 2.3 Тема (Topic)

**Определение:** Тема — это минимальная единица обучения внутри раздела, которая может содержать теоретический материал и/или тест.

**Характеристики:**

- Название темы
- Порядковый номер в разделе
- Теоретический материал (опционально)
- Тест темы (опционально)

**Примеры тем для раздела "Пицца":**

- Тема 1: "Как раскатать пиццу"
- Тема 2: "Как работает печь"
- Тема 3: "Как собрать пиццу Семейная"
- Тема 4: "Как готовить соус для пиццы"

**Важно:** Тема может содержать:

- Только материал (без теста)
- Только тест (без материала)
- И материал, и тест одновременно

### 2.4 Аттестация (Assessment)

**Определение:** Аттестация — это независимая сущность для проверки знаний, которая может существовать как внутри курса, так и отдельно.

**Типы аттестаций:**

1. **Тест темы** — проверка знаний по конкретной теме (опционально)
2. **Тест раздела** — общий тест по всему разделу (обязательно для завершения раздела)
3. **Итоговая аттестация курса** — финальная проверка знаний по всему курсу (опционально, настраивается администратором)
4. **Standalone-аттестация** — независимый тест, не привязанный к курсу (для переаттестации сотрудников)

**Характеристики:**

- Название аттестации
- Вопросы и варианты ответов
- Проходной балл (процент правильных ответов)
- Максимальное количество попыток
- Ограничение по времени

## 3. СТРУКТУРА КУРСА

### 3.1 Иерархия

```
КУРС "Обучение поваров"
│
├─ РАЗДЕЛ 1: "Пицца" (обязательный)
│  ├─ Тема 1: "Как раскатать пиццу" [материал + тест]
│  ├─ Тема 2: "Как работает печь" [материал + тест]
│  ├─ Тема 3: "Как собрать пиццу Семейная" [материал + тест]
│  ├─ Тема 4: "Как готовить соус" [только материал]
│  └─ ТЕСТ РАЗДЕЛА "Пицца"
│
├─ РАЗДЕЛ 2: "Закуски" (обязательный)
│  ├─ Тема 1: "Салаты" [материал + тест]
│  ├─ Тема 2: "Горячие закуски" [только материал]
│  └─ ТЕСТ РАЗДЕЛА "Закуски"
│
├─ РАЗДЕЛ 3: "Заготовка" (необязательный)
│  ├─ Тема 1: "Подготовка ингредиентов" [только материал]
│  └─ ТЕСТ РАЗДЕЛА "Заготовка"
│
└─ ИТОГОВАЯ АТТЕСТАЦИЯ КУРСА (опционально)
```

### 3.2 Минимальные требования

**Для курса:**

- Минимум 1 раздел
- Итоговая аттестация — опционально (настраивается администратором)

**Для раздела:**

- Минимум 1 тема
- Обязательно должен быть тест раздела (общий для всех тем)
- Может быть обязательным или необязательным

**Для темы:**

- Может содержать только материал (без теста)
- Может содержать только тест (без материала)
- Может содержать и материал, и тест
- Минимум: хотя бы что-то одно (материал или тест)

### 3.3 Типы разделов

**Обязательный раздел:**

- Должен быть пройден для завершения курса
- Блокирует доступ к следующим разделам до своего завершения
- Необходим для открытия итоговой аттестации курса

**Необязательный раздел:**

- Может быть пропущен пользователем
- Не блокирует доступ к следующим разделам
- Не требуется для открытия итоговой аттестации курса
- Не влияет на статус завершения курса

## 4. ЛОГИКА ПРОХОЖДЕНИЯ

### 4.1 Последовательность прохождения тем

**Правило:** Темы внутри раздела проходятся **строго по порядку**.

**Механизм блокировки:**

1. При старте раздела открыта только первая тема
2. Следующая тема открывается только после успешного завершения предыдущей
3. Пользователь не может перейти к теме N+1, пока не завершена тема N

**Пример:**

```
Раздел "Пицца":
  ✅ Тема 1: "Как раскатать пиццу" — доступна сразу
  🔒 Тема 2: "Как работает печь" — заблокирована до завершения Темы 1
  🔒 Тема 3: "Как собрать пиццу" — заблокирована до завершения Темы 2
  🔒 Тема 4: "Как готовить соус" — заблокирована до завершения Темы 3
```

### 4.2 Условия завершения темы

Тема считается завершённой, когда выполнены **все** следующие условия:

1. **Если у темы есть материал:** пользователь открыл и просмотрел материал
2. **Если у темы есть тест:** пользователь прошёл тест с результатом выше проходного балла

**Примеры:**

| Тип темы        | Условие завершения                    |
| --------------- | ------------------------------------- |
| Только материал | Просмотреть материал                  |
| Только тест     | Сдать тест (≥ проходной балл)         |
| Материал + тест | Просмотреть материал **И** сдать тест |

### 4.3 Условия завершения раздела

Раздел считается завершённым, когда выполнены **все** следующие условия:

1. Все темы внутри раздела успешно завершены
2. Сдан тест раздела с результатом ≥ проходного балла

**Важно:** Даже если все темы пройдены, раздел не считается завершённым до сдачи теста раздела.

### 4.4 Последовательность прохождения разделов

**Правило:** Следующий раздел открывается **автоматически** после успешного завершения предыдущего.

**Механизм:**

1. При старте курса открыт только первый раздел
2. После завершения раздела N автоматически открывается раздел N+1
3. Необязательные разделы не блокируют прохождение — их можно пропустить

**Пример:**

```
✅ Раздел 1: "Пицца" (обязательный) — завершён
   → автоматически открылся
✅ Раздел 2: "Закуски" (обязательный) — доступен
🔒 Раздел 3: "Заготовка" (необязательный) — заблокирован до завершения Раздела 2
```

### 4.5 Доступ к итоговой аттестации курса

**Правило:** Итоговая аттестация курса открывается, когда **все обязательные разделы** пройдены успешно.

**Важно:**

- Необязательные разделы НЕ учитываются для открытия итоговой аттестации
- Если итоговой аттестации нет (не настроена администратором), она не требуется для завершения курса

### 4.6 Условия завершения курса

Курс считается **полностью завершённым**, когда выполнены следующие условия:

1. Все **обязательные** разделы пройдены успешно
2. Если у курса есть итоговая аттестация — она сдана с результатом ≥ проходного балла

**Важно:** Необязательные разделы **не влияют** на статус завершения курса.

**Примеры:**

| Ситуация                                                       | Статус курса   |
| -------------------------------------------------------------- | -------------- |
| Все обязательные разделы пройдены + итоговая аттестация сдана  | ✅ Завершён    |
| Все обязательные разделы пройдены + итоговой аттестации нет    | ✅ Завершён    |
| Все обязательные разделы пройдены + необязательные не пройдены | ✅ Завершён    |
| Хотя бы один обязательный раздел не пройден                    | ❌ Не завершён |

### 4.7 Возврат к пройденным материалам

**Правило:** Пользователь может **в любой момент** вернуться к уже пройденной теме и пересмотреть материал.

**Возможности:**

- Повторный просмотр теоретических материалов
- Повторная сдача тестов (в пределах лимита попыток)
- Доступ сохраняется даже после завершения всего курса

## 5. НАЗНАЧЕНИЕ КУРСОВ

### 5.1 Способы назначения

Курсы назначаются пользователям **комбинированным способом**:

1. **Автоматическое назначение по должности**
   - Администратор указывает целевые должности для курса
   - Все сотрудники с указанными должностями автоматически получают доступ к курсу
2. **Автоматическое назначение по филиалу**
   - Администратор указывает целевые филиалы для курса
   - Все сотрудники указанных филиалов автоматически получают доступ к курсу

3. **Ручное назначение**
   - Администратор может вручную назначить курс конкретному пользователю
   - Это позволяет гибко управлять доступом вне зависимости от должности/филиала

### 5.2 Логика комбинирования

Пользователь получает доступ к курсу, если выполняется **хотя бы одно** из условий:

- Должность пользователя входит в список целевых должностей курса
- **ИЛИ** филиал пользователя входит в список целевых филиалов курса
- **ИЛИ** курс назначен пользователю вручную администратором

### 5.3 Примеры назначения

**Пример 1: Курс "Обучение поваров"**

- Целевые должности: Повар, Су-шеф
- Целевые филиалы: не указаны
- Ручные назначения: нет

**Результат:** Все повара и су-шефы из всех филиалов видят этот курс.

**Пример 2: Курс "Стандарты обслуживания — Москва"**

- Целевые должности: Официант, Администратор зала
- Целевые филиалы: Москва (все филиалы)
- Ручные назначения: Иван Петров (курьер)

**Результат:**

- Все официанты и администраторы из московских филиалов видят курс
- Курьер Иван Петров тоже видит курс (ручное назначение)

## 6. СИСТЕМА АТТЕСТАЦИЙ

### 6.1 Независимость аттестаций

**Ключевой принцип:** Аттестации — это **независимые сущности**, которые могут существовать как внутри курсов, так и отдельно.

**Два контекста использования:**

1. **Аттестация внутри курса**
   - Используется в темах (тест темы)
   - Используется в разделах (тест раздела)
   - Используется как итоговая аттестация курса

2. **Standalone-аттестация**
   - Существует независимо от курсов
   - Используется для переаттестации сотрудников
   - Позволяет проверять знания без повторного прохождения всего курса

### 6.2 Типы аттестаций в курсе

**1. Тест темы (опционально)**

- Проверяет знания по одной конкретной теме
- Настраивается индивидуально для каждой темы
- Может отсутствовать, если тема содержит только материал

**2. Тест раздела (обязательно)**

- Проверяет знания по всем темам раздела
- Обязателен для каждого раздела
- Сдаётся после прохождения всех тем раздела

**3. Итоговая аттестация курса (опционально)**

- Проверяет знания по всему курсу
- Может включать вопросы из всех разделов и тем
- Настраивается администратором при создании курса

### 6.3 Настройки аттестации

**Проходной балл:**

- Минимальный процент правильных ответов для успешной сдачи
- Настраивается для каждой аттестации отдельно
- Пример: 70% — нужно ответить правильно на 7 из 10 вопросов

**Максимальное количество попыток:**

- Ограничение на количество попыток сдачи аттестации
- Настраивается для каждой аттестации отдельно
- Пример: 3 попытки — после трёх провалов тест больше недоступен

**Ограничение по времени:**

- Время, отведённое на прохождение аттестации
- Настраивается для каждой аттестации отдельно
- Пример: 30 минут на тест из 20 вопросов

### 6.4 Пересдача аттестаций

**Правило:** Если пользователь провалил тест (не набрал проходной балл), он может пересдать его в пределах установленного лимита попыток.

**Механизм:**

1. После провала аттестации пользователь видит количество оставшихся попыток
2. Пользователь может немедленно начать новую попытку
3. После исчерпания всех попыток тест становится недоступен
4. Для разблокировки требуется вмешательство администратора

**Важно:** Пользователь может вернуться к материалу темы для повторного изучения перед пересдачей.

### 6.5 Standalone-аттестации (переаттестация)

**Назначение:** Периодическая проверка знаний сотрудников без повторного прохождения всего курса.

**Сценарий использования:**

1. Сотрудник прошёл курс "Обучение поваров" полгода назад
2. Для проверки актуальности знаний ему назначается standalone-аттестация "Переаттестация поваров"
3. Сотрудник сдаёт только тест, не проходя заново весь курс

**Отличия от курсовых аттестаций:**

- Не требует прохождения материалов
- Не привязана к конкретному курсу
- Может быть назначена в любой момент
- Может содержать вопросы из разных курсов

## 7. ПРОГРЕСС ПОЛЬЗОВАТЕЛЯ

### 7.1 Отслеживание прогресса

Система отслеживает прогресс пользователя на трёх уровнях:

1. **Прогресс по курсу в целом**
   - Процент завершения курса (0-100%)
   - Количество пройденных разделов
   - Количество пройденных тем
   - Статус итоговой аттестации

2. **Прогресс по разделу**
   - Статус раздела (заблокирован / доступен / в процессе / завершён)
   - Процент завершения раздела
   - Количество пройденных тем в разделе

3. **Прогресс по теме**
   - Статус темы (заблокирована / доступна / в процессе / завершена / провалена)
   - Просмотрен ли материал
   - Результат теста темы
   - Количество использованных попыток

### 7.2 Статусы курса

| Статус     | Описание                          | Условие                                                                    |
| ---------- | --------------------------------- | -------------------------------------------------------------------------- |
| Не начат   | Пользователь ещё не открывал курс | Нет прогресса                                                              |
| В процессе | Пользователь проходит курс        | Есть прогресс, но не все обязательные разделы завершены                    |
| Завершён   | Курс пройден полностью            | Все обязательные разделы завершены + итоговая аттестация сдана (если есть) |

### 7.3 Статусы раздела

| Статус       | Описание                     | Условие                                                         |
| ------------ | ---------------------------- | --------------------------------------------------------------- |
| Заблокирован | Раздел недоступен            | Предыдущий раздел не завершён                                   |
| Доступен     | Раздел можно начать          | Предыдущий раздел завершён, текущий не начат                    |
| В процессе   | Раздел начат, но не завершён | Есть прогресс, но не все темы пройдены или тест раздела не сдан |
| Завершён     | Раздел пройден полностью     | Все темы завершены + тест раздела сдан                          |

### 7.4 Статусы темы

| Статус        | Описание                     | Условие                                                 |
| ------------- | ---------------------------- | ------------------------------------------------------- |
| Заблокирована | Тема недоступна              | Предыдущая тема не завершена                            |
| Доступна      | Тему можно начать            | Предыдущая тема завершена, текущая не начата            |
| В процессе    | Тема начата, но не завершена | Материал просмотрен, но тест не сдан (или наоборот)     |
| Завершена     | Тема пройдена успешно        | Материал просмотрен (если есть) + тест сдан (если есть) |
| Провалена     | Исчерпаны все попытки теста  | Пользователь провалил тест и попытки закончились        |

### 7.5 Расчёт процента завершения

**Процент завершения курса** рассчитывается по формуле:

```
Процент = (Количество завершённых обязательных разделов / Общее количество обязательных разделов) × 100%
```

**Важно:** Необязательные разделы не учитываются в общем проценте завершения курса.

**Процент завершения раздела** рассчитывается по формуле:

```
Процент = (Количество завершённых тем / Общее количество тем в разделе) × 100%
```

**Важно:** Тест раздела также считается как одна "тема" для расчёта процента.

## 8. АДМИНИСТРАТИВНЫЕ ФУНКЦИИ

### 8.1 Создание курса

Администратор может создать новый курс с следующими параметрами:

**Обязательные поля:**

- Название курса
- Описание курса

**Опциональные поля:**

- Целевые должности (для автоматического назначения)
- Целевые филиалы (для автоматического назначения)
- Итоговая аттестация курса (выбор из списка существующих аттестаций)

**Статус:** Новый курс создаётся в статусе "Черновик".

### 8.2 Создание раздела

Администратор добавляет разделы в курс:

**Обязательные поля:**

- Название раздела
- Порядковый номер в курсе

**Опциональные поля:**

- Описание раздела
- Обязательность (обязательный / необязательный)

**Важно:** Каждый раздел должен иметь тест раздела.

### 8.3 Создание темы

Администратор добавляет темы в раздел:

**Обязательные поля:**

- Название темы
- Порядковый номер в разделе

**Опциональные поля:**

- Теоретический материал (текст, видео, ссылки, документы)
- Тест темы (выбор из списка существующих аттестаций)

**Важно:** Тема должна содержать хотя бы что-то одно — материал или тест.

### 8.4 Назначение аттестаций

Администратор может назначить аттестации в трёх местах:

1. **Тест темы** — выбирается при создании/редактировании темы
2. **Тест раздела** — выбирается при создании/редактировании раздела
3. **Итоговая аттестация курса** — выбирается в настройках курса

**Важно:** Одну и ту же аттестацию можно использовать в разных темах/разделах/курсах.

### 8.5 Публикация курса

Перед публикацией система проверяет:

1. Курс содержит минимум 1 раздел
2. Каждый раздел содержит минимум 1 тему
3. Каждая тема содержит материал или тест
4. У каждого раздела назначен тест раздела

Если все проверки пройдены — курс публикуется и становится доступен пользователям.

### 8.6 Архивирование курса

Администратор может архивировать курс:

- Архивированный курс скрывается из каталога для новых пользователей
- Пользователи, уже начавшие курс, могут продолжить его прохождение
- Новые пользователи не могут начать архивированный курс

### 8.7 Ручное назначение курса

Администратор может вручную назначить курс конкретному пользователю:

1. Выбрать пользователя из списка
2. Выбрать курс из списка
3. Назначить курс

Пользователь сразу получает доступ к курсу.

### 8.8 Просмотр прогресса пользователей

Администратор может просматривать:

- Список всех пользователей, зачисленных на курс
- Прогресс каждого пользователя (процент завершения, текущий раздел/тема)
- Результаты аттестаций (баллы, количество попыток, дата прохождения)
- Время, затраченное на прохождение курса

### 8.9 Сброс прогресса

Администратор может сбросить прогресс пользователя по курсу:

- Весь прогресс обнуляется
- Пользователь начинает курс заново с первого раздела
- История попыток аттестаций сохраняется для статистики

## 9. ПОЛЬЗОВАТЕЛЬСКИЕ ФУНКЦИИ

### 9.1 Просмотр доступных курсов

Пользователь видит список курсов, назначенных ему:

- По должности
- По филиалу
- Вручную администратором

Для каждого курса отображается:

- Название и описание
- Прогресс прохождения (если курс начат)
- Количество разделов и тем

### 9.2 Начало курса

При первом входе в курс:

1. Система фиксирует дату начала курса
2. Открывается доступ к первому разделу
3. Остальные разделы блокируются
4. Создаётся запись о прогрессе пользователя

### 9.3 Прохождение темы

**Шаг 1: Просмотр материала**

- Пользователь открывает тему
- Просматривает теоретический материал (текст, видео, документы)
- Система фиксирует факт просмотра

**Шаг 2: Прохождение теста (если есть)**

- Пользователь начинает тест темы
- Отвечает на вопросы
- Отправляет ответы
- Получает результат (баллы, процент правильных ответов)

**Шаг 3: Переход к следующей теме**

- Если тема завершена успешно — открывается следующая тема
- Если тест провален — пользователь может пересдать (в пределах попыток)

### 9.4 Прохождение раздела

После завершения всех тем раздела:

1. Открывается доступ к тесту раздела
2. Пользователь проходит тест раздела
3. Если тест сдан успешно — раздел завершён, открывается следующий раздел
4. Если тест провален — пользователь может пересдать (в пределах попыток)

### 9.5 Прохождение итоговой аттестации

После завершения всех обязательных разделов:

1. Открывается доступ к итоговой аттестации курса (если она настроена)
2. Пользователь проходит итоговую аттестацию
3. Если аттестация сдана успешно — курс завершён
4. Если аттестация провалена — пользователь может пересдать (в пределах попыток)

### 9.6 Возврат к пройденным материалам

Пользователь может в любой момент:

- Вернуться к любой пройденной теме
- Повторно просмотреть материалы
- Повторно пройти тест (в пределах попыток)
- Просмотреть свои результаты

### 9.7 Отслеживание прогресса

Пользователь может видеть:

- Общий процент завершения курса
- Список всех разделов с их статусами
- Текущий раздел и тему
- Количество оставшихся попыток для каждого теста

## 10. БИЗНЕС-ПРАВИЛА И ОГРАНИЧЕНИЯ

### 10.1 Правила последовательности

1. **Темы проходятся строго по порядку** — нельзя перейти к теме N+1, пока не завершена тема N
2. **Разделы открываются последовательно** — следующий раздел доступен только после завершения предыдущего
3. **Необязательные разделы можно пропускать** — они не блокируют прохождение курса
4. **Итоговая аттестация доступна только после всех обязательных разделов**

### 10.2 Правила завершения

1. **Тема завершена** = материал просмотрен (если есть) + тест сдан (если есть)
2. **Раздел завершён** = все темы завершены + тест раздела сдан
3. **Курс завершён** = все обязательные разделы завершены + итоговая аттестация сдана (если есть)

### 10.3 Правила попыток

1. **Количество попыток ограничено** — настраивается для каждой аттестации
2. **После исчерпания попыток** — тест недоступен, требуется вмешательство администратора
3. **Попытки считаются независимо** — для теста темы, теста раздела и итоговой аттестации отдельно

### 10.4 Правила назначения

1. **Автоназначение по должности** — все сотрудники указанных должностей получают доступ
2. **Автоназначение по филиалу** — все сотрудники указанных филиалов получают доступ
3. **Ручное назначение** — администратор может назначить курс любому пользователю
4. **Комбинирование** — если выполнено хотя бы одно условие, пользователь получает доступ

### 10.5 Правила независимости аттестаций

1. **Аттестация — отдельная сущность** — не привязана к конкретному курсу
2. **Переиспользование** — одну аттестацию можно использовать в разных темах/разделах/курсах
3. **Standalone-аттестации** — могут существовать отдельно для переаттестации

### 10.6 Правила обязательности

1. **Обязательный раздел** — должен быть пройден для завершения курса
2. **Необязательный раздел** — может быть пропущен, не влияет на завершение курса
3. **Итоговая аттестация** — опциональна, настраивается администратором

### 10.7 Правила статусов

1. **Черновик** — курс редактируется, недоступен пользователям
2. **Опубликован** — курс доступен пользователям
3. **Архивирован** — курс скрыт для новых пользователей, но доступен для тех, кто начал

### 10.8 Ограничения структуры

1. **Минимум 1 раздел** — курс не может быть опубликован без разделов
2. **Минимум 1 тема в разделе** — раздел не может быть пустым
3. **Материал или тест** — каждая тема должна содержать хотя бы что-то одно
4. **Обязательный тест раздела** — каждый раздел должен иметь тест

## ГЛОССАРИЙ

**Курс** — структурированная программа обучения из разделов и тем

**Раздел** — тематическая группа тем внутри курса

**Тема** — минимальная единица обучения (материал + опциональный тест)

**Аттестация** — независимая сущность для проверки знаний (тест)

**Тест темы** — аттестация, проверяющая знания по одной теме

**Тест раздела** — обязательная аттестация, проверяющая знания по всему разделу

**Итоговая аттестация** — опциональная аттестация, проверяющая знания по всему курсу

**Standalone-аттестация** — независимый тест, не привязанный к курсу (для переаттестации)

**Обязательный раздел** — раздел, который должен быть пройден для завершения курса

**Необязательный раздел** — раздел, который можно пропустить

**Проходной балл** — минимальный процент правильных ответов для успешной сдачи теста

**Попытка** — одна попытка прохождения аттестации

**Прогресс** — процент завершения курса/раздела/темы
