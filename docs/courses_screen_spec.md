# SCREEN: ADMIN — Список курсов {#screen-admin-courses-list}

## 1. НАЗНАЧЕНИЕ

Главная страница раздела «Курсы» в административной панели. Отображает все курсы в системе, позволяет фильтровать, переходить к редактированию, публиковать, закрывать и удалять курсы. Внизу страницы — аналитическая сводка и воронка прохождения.

---

## 2. ЦЕЛЬ ПОЛЬЗОВАТЕЛЯ

Быстро найти нужный курс, оценить его текущий статус, выполнить целевое действие (публикация / закрытие / удаление) или перейти к редактированию.

---

## 3. ОПИСАНИЕ LAYOUT

**Верхняя зона (page-header)**  
Кнопка «Создать курс» с иконкой «+» — единственный элемент. Выровнена по правому краю [ПРЕДПОЛОЖЕНИЕ по стандарту admin-panel].

**Зона фильтров (filters-card)**  
Карточка с горизонтальной сеткой из 4 элементов:

- Поле поиска (текст)
- Селект статуса
- Кнопка «Найти»
- Кнопка «Сбросить»

**Зона таблицы / мобильных карточек (courses-card)**  
Основной контент. На десктопе — таблица, на мобильном — стек карточек (CSS-классы `hide-mobile` / `show-mobile`).

**Зона аналитики**

- Сводка метрик (`AnalyticsSummaryCards`) — всегда видна, показывает заглушку до подключения реальных данных [ФАКТ: комментарий в коде].
- Воронка курсов (`funnel-card`) — видна только при наличии данных (`v-if="funnel.length > 0"`).

---

## 4. UI-ЭЛЕМЕНТЫ

### Блок фильтров

| Элемент           | Тип                                   | Назначение                                                |
| ----------------- | ------------------------------------- | --------------------------------------------------------- |
| Поиск             | Input                                 | Поиск по названию и описанию курса                        |
| Статус            | Select                                | Фильтр по статусу (Все / Черновик / Опубликован / Закрыт) |
| Кнопка «Найти»    | Button secondary + иконка search      | Запускает запрос к API                                    |
| Кнопка «Сбросить» | Button secondary + иконка refresh-ccw | Сбрасывает фильтры и перезагружает список                 |

### Таблица курсов (десктоп)

| Колонка   | Содержимое                                                                                              |
| --------- | ------------------------------------------------------------------------------------------------------- |
| Название  | Заголовок (жирный), подпись «Есть неопубликованные изменения» (если `hasDraft`), описание (серый текст) |
| Статус    | Badge с цветовым вариантом по статусу                                                                   |
| Тем курса | Число (`sectionsCount`)                                                                                 |
| Версия    | Версионный номер курса                                                                                  |
| Обновлен  | Дата последнего обновления (форматированная)                                                            |
| Действия  | Набор кнопок, зависящий от статуса курса                                                                |

### Кнопки действий в строке (логика отображения)

| Условие                  | Кнопки                                |
| ------------------------ | ------------------------------------- |
| Любой статус             | «Изменить»                            |
| `published` + `hasDraft` | «Открыть черновик»                    |
| `draft`                  | «Опубликовать» (success, иконка send) |
| `published`              | «Закрыть» (secondary, иконка archive) |
| Не `published`           | «Удалить» (danger, иконка trash)      |

### Мобильные карточки

Содержат те же поля, что и таблица, но в вертикальном порядке. Кнопки действий растянуты на всю ширину (`fullWidth`).

### Сводка аналитики (AnalyticsSummaryCards)

[ФАКТ] Компонент подключён, но содержит заглушку — реальные метрики ещё не подключены.

### Воронка курсов (таблица)

| Колонка        | Описание                                |
| -------------- | --------------------------------------- |
| Курс           | Название                                |
| Назначено      | Количество пользователей с назначением  |
| Начали         | Перешли в статус started                |
| Проходят       | Активны (in_progress)                   |
| Завершили      | Прошли курс полностью                   |
| Попытки (темы) | Суммарные попытки тестов по темам       |
| Попытки (итог) | Попытки итоговой аттестации             |
| Время в курсе  | Форматированная суммарная длительность  |
| Ср. балл курса | Средний балл по всем тестам тем         |
| Ср. балл итога | Средний балл итоговой аттестации        |
| Ср. прогресс   | Средний % прогресса                     |
| Конверсия      | `completedCount / assignedCount * 100%` |

---

## 5. ВЗАИМОДЕЙСТВИЯ

- Поиск запускается кнопкой «Найти» или клавишей `Enter` в поле поиска.
- Изменение селекта статуса немедленно перезагружает список (`@update:modelValue="loadCourses"`).
- Клик «Изменить» / «Открыть черновик» → переход в редактор.
- Клик «Опубликовать» → вызов `handlePublish`, затем перезагрузка списка.
- Клик «Закрыть» → вызов `handleArchive`.
- Клик «Удалить» → вызов `handleDelete` [ПРЕДПОЛОЖЕНИЕ: с подтверждением, стандартная практика].
- Клик «Создать курс» → переход на `CourseEditorView` в режиме создания.

---

## 6. СОСТОЯНИЯ

| Состояние         | Описание                                                           |
| ----------------- | ------------------------------------------------------------------ |
| Loading           | Preloader поверх карточки таблицы                                  |
| Empty             | Текст «Курсы не найдены»                                           |
| Filled (десктоп)  | Таблица со строками                                                |
| Filled (мобайл)   | Стек карточек                                                      |
| Строка с hasDraft | Отображается метка «Есть неопубликованные изменения» под названием |

### Состояния Badge статуса

| Статус          | Вариант                                                |
| --------------- | ------------------------------------------------------ |
| draft           | warning (жёлтый) [ПРЕДПОЛОЖЕНИЕ по `getStatusVariant`] |
| published       | success (зелёный)                                      |
| archived/closed | secondary/neutral                                      |

---

## 7. БИЗНЕС-ПРАВИЛА

- Опубликованный курс нельзя удалить.
- Черновик можно удалить.
- Если у опубликованного курса есть черновик (`hasDraft=true`), отображается кнопка «Открыть черновик» рядом с «Изменить».
- Воронка показывается только при наличии данных.
- Сводка аналитики — заглушка, не несёт реальных данных.

---

## 8. ВАЛИДАЦИИ И ОШИБКИ

- [ПРЕДПОЛОЖЕНИЕ] «Удалить» — должно запрашивать подтверждение. В коде явного диалога не видно — риск случайного удаления.
- Ошибки операций (publish / archive / delete) отображаются через `useToast`.

---

## 9. НАВИГАЦИЯ

- Кнопка «Создать курс» → `/courses/new` (или аналогичный роут).
- Кнопки «Изменить» / «Открыть черновик» → `/courses/:id`.

---

## 10. РИСКИ И НЕЯСНОСТИ

1. **Нет подтверждения удаления** — прямой вызов `handleDelete` без явного confirmation dialog в шаблоне. Критический UX-риск.
2. **Аналитика — заглушка** — пользователь видит блок сводки, но данные не реальные. Вводит в заблуждение.
3. **Отсутствует пагинация** — при большом числе курсов список вырастет бесконечно. Нет ни lazy load, ни постраничной навигации.
4. **Воронка не интерактивна** — нет возможности кликнуть на строку и перейти к детальному отчёту по курсу.
5. **Два дублирующихся действия** — при `published + hasDraft` показываются и «Изменить», и «Открыть черновик», ведущие на один и тот же экран. Различие неочевидно пользователю.
6. **Колонка «Тем курса»** — некорректное название в терминологии (бизнес-термин: «Тем курса», тогда как внутри редактора используется «Темы курса» и «Подтемы»).

---

## 11. РЕКОМЕНДАЦИИ

1. Добавить confirmation dialog для удаления курса.
2. Убрать блок аналитики или явно пометить его «Данные временно недоступны».
3. Ввести пагинацию или виртуальный скролл.
4. При `published + hasDraft` показывать только одну кнопку «Редактировать черновик» — семантически точнее.
5. Добавить возможность клика на строку воронки для перехода к детальной аналитике курса.
6. Добавить сортировку по колонкам таблицы (хотя бы по дате и статусу).

# SCREEN: ADMIN — Редактор курса, Шаг 1. Основная информация {#screen-admin-editor-step1}

## 1. НАЗНАЧЕНИЕ

Экран создания нового курса и редактирования существующего. Реализован как 4-шаговый мастер (wizard). Шаг 1 — заполнение базовых атрибутов курса.

---

## 2. ЦЕЛЬ ПОЛЬЗОВАТЕЛЯ

Заполнить обязательные поля (название) и опциональные (описание, обложка, категория, теги, срок действия), сохранить черновик курса, затем перейти к наполнению структурой.

---

## 3. ОПИСАНИЕ LAYOUT

**Верхняя зона (page-header)**

- Кнопка «К списку курсов» (secondary, иконка arrow-left) — слева.
- В режиме редактирования справа: `DraftSaveIndicator` + Badge статуса + кнопки управления публикацией.

**Мастер-карточка (wizard-card)**

- Заголовок «Мастер настройки курса» + подпись (различается для режима создания и редактирования).
- Badge «Шаг X из 4» — справа в строке заголовка.
- Строка шагов: 4 индикатора с номером и подписью; активный выделен, пройденные — помечены как completed.

**Карточка контента шага 1 (editor-card)**

- Заголовок «Шаг 1. Основная информация» + подпись.
- Двухколоночная сетка полей.

**Карточка назначения (assignments-card)** [ФАКТ: видна только на Шаге 1 в режиме редактирования]

- Секция «Целевые должности» (checkbox-list).
- Секция «Целевые филиалы» (checkbox-list).
- Кнопка «Сохранить назначения».
- Таблица ручных назначений.

**Навигационная строка мастера (wizard-navigation)**

- Кнопка «Назад» (если шаг > 1).
- Кнопка «Далее» (если шаг < 4, с блокировкой при `!canProceed`).
- На шаге 4: кнопки публикации / закрытия.

---

## 4. UI-ЭЛЕМЕНТЫ (Шаг 1)

### Поля основной информации

| Поле              | Тип                       | Обязательно | Описание                                                   |
| ----------------- | ------------------------- | ----------- | ---------------------------------------------------------- |
| Название курса    | Input text                | Да          | Placeholder «Стандарты обслуживания»                       |
| Статус публикации | Badge (readonly)          | —           | Текущий статус; в режиме создания — «Черновик»             |
| Описание курса    | Textarea (4 строки)       | Нет         | Span на всю ширину сетки (grid-span-full)                  |
| Ссылка на обложку | Input text                | Нет         | URL изображения                                            |
| Загрузить файл    | input[type=file] + Button | Нет         | Кнопка заблокирована до сохранения курса                   |
| Подсказка обложки | Текст                     | —           | «Для загрузки файла курс должен быть сначала сохранён.»    |
| Категория         | Input text                | Нет         | Произвольная строка, placeholder «Сервис»                  |
| Теги              | Input text                | Нет         | Строка через запятую, span на всю ширину                   |
| Срок действия     | Select                    | Нет         | 3 варианта: Бессрочный / N дней / Фиксированные даты       |
| Количество дней   | Input number              | Условно     | Показывается только при `availabilityMode = relative_days` |
| Дата начала       | Input datetime-local      | Условно     | Показывается только при `availabilityMode = fixed_dates`   |
| Дата окончания    | Input datetime-local      | Условно     | Показывается только при `availabilityMode = fixed_dates`   |

### DraftSaveIndicator

Компонент в шапке (только режим редактирования). Отображает состояние автосохранения черновика: сохраняется / сохранён / ошибка + время последнего сохранения.

### Блок назначения (только редактирование, Шаг 1)

**Правило видимости:** если ни одна должность и ни один филиал не выбраны — курс виден всем пользователям [ФАКТ: комментарий в коде].

| Элемент                    | Тип                           | Назначение                                                                                                              |
| -------------------------- | ----------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| Список должностей          | checkbox-list                 | Выбор целевых должностей                                                                                                |
| Список филиалов            | checkbox-list                 | Выбор целевых филиалов                                                                                                  |
| «Сохранить назначения»     | Button                        | Сохраняет выбор (saveTargets)                                                                                           |
| Поле ID пользователя       | Input number                  | Для ручного назначения                                                                                                  |
| Индивидуальный дедлайн     | Input datetime-local          | Персональный дедлайн пользователя                                                                                       |
| «Добавить»                 | Button                        | Создаёт ручное назначение (handleAddAssignment)                                                                         |
| Таблица ручных назначений  | Table                         | Показывает назначенных пользователей с колонками: имя, должность, филиал, кем назначен, дата, дедлайн, статус, действия |
| Кнопка закрытия назначения | Button secondary icon:archive | Закрывает назначение пользователя                                                                                       |
| Кнопка удаления назначения | Button danger icon:trash      | Удаляет запись о назначении                                                                                             |

---

## 5. ВЗАИМОДЕЙСТВИЯ

- Выбор `availabilityMode` динамически показывает / скрывает поля дат и количества дней.
- Клик по `input[type=file]` → выбор файла → кнопка «Загрузить файл» активируется (если курс сохранён).
- Нажатие «Далее» → переход к шагу 2 (если `canProceed = true`).
- Нажатие «Сохранить назначения» → API-запрос, toast-уведомление.
- Добавление ручного назначения → немедленное добавление в таблицу.
- Клик закрыть / удалить назначение → локальный loading на кнопке, API-запрос.

---

## 6. СОСТОЯНИЯ

| Элемент                      | Состояние              | Описание                                     |
| ---------------------------- | ---------------------- | -------------------------------------------- |
| Кнопка «Загрузить файл»      | disabled               | Курс ещё не сохранён                         |
| Кнопка «Далее»               | disabled               | `canProceed = false` (название не заполнено) |
| Кнопка «Опубликовать»        | loading                | Идёт запрос публикации                       |
| Кнопка «Закрыть курс»        | loading                | Идёт запрос архивации                        |
| DraftSaveIndicator           | saving / saved / error | Состояние автосохранения                     |
| Список должностей / филиалов | Загрузка               | «Загрузка...» при пустом массиве             |

---

## 7. БИЗНЕС-ПРАВИЛА

- Загрузка обложки возможна только после первичного сохранения курса.
- Если не выбраны ни должности, ни филиалы — курс назначается всем.
- Поля срока действия показываются условно по выбранному режиму.
- Кнопка «Опубликовать изменения» показывается в шапке только при `hasDraftChanges = true`.

---

## 8. ВАЛИДАЦИИ И ОШИБКИ

- Поле «Название» обязательно (`required`). При попытке перейти на шаг 2 без него — `canProceed = false`, кнопка «Далее» заблокирована.
- Поле «Название» показывает inline-ошибку (`errors.title`).
- Количество дней: `min="1"`, `max="3650"`.
- ID пользователя: `min="1"`, тип number.

---

## 9. НАВИГАЦИЯ

- «К списку курсов» → `/courses`.
- «Далее» → шаг 2 (только если курс уже сохранён, т.е. `isEditMode`).
- Кнопки в шапке (Опубликовать / Отменить черновик / Закрыть курс) доступны в любой момент в режиме редактирования.

---

## 10. РИСКИ И НЕЯСНОСТИ

1. **Нет явного шага «Сохранить»** — при создании нового курса пользователь должен нажать «Далее», чтобы курс сохранился, но UI этого не объясняет явно. Подпись «Сначала сохраните основу курса...» есть, но она малозаметна.
2. **Поле «Теги» — строка через запятую** — нет никакой визуальной обработки (chip/tag), введённый текст хранится как строка. Пользователь не знает формат ввода.
3. **Ручное назначение по ID** — ввод ID пользователя вручную неудобен, нет автодополнения / поиска по имени.
4. **Нет ограничения по дедлайну** — можно поставить дедлайн в прошлом без ошибки [ПРЕДПОЛОЖЕНИЕ].
5. **Срок действия курса и индивидуальный дедлайн** — два разных механизма ограничения. Их приоритет и взаимодействие не описаны в UI.

---

## 11. РЕКОМЕНДАЦИИ

1. Добавить явную кнопку «Сохранить и продолжить» вместо использования «Далее» как сохранения.
2. Поле «Теги» переделать в chip-input с разбивкой по запятой.
3. Поиск пользователей для ручного назначения — input с автодополнением по имени/телефону вместо ввода ID.
4. Добавить валидацию дедлайна (не в прошлом).
5. Добавить tooltip или подсказку про приоритет срока действия vs. индивидуального дедлайна.

# SCREEN: ADMIN — Редактор курса, Шаг 2. Темы и подтемы {#screen-admin-editor-step2}

## 1. НАЗНАЧЕНИЕ

Наполнение структуры курса: создание тем курса (sections), вложенных подтем (topics), привязка тестов к темам и подтемам, настройка порядка и обязательности.

---

## 2. ЦЕЛЬ ПОЛЬЗОВАТЕЛЯ

Добавить все темы курса, внутри каждой — подтемы, настроить порядок, обязательность и прикрепить тесты.

---

## 3. ОПИСАНИЕ LAYOUT

## 7. БИЗНЕС-ПРАВИЛА

- Порядок сохраняется автоматически после drag-end или нажатия стрелок.

- Название темы/подтемы: обязательное, инлайн-ошибка при пустом значении.
- Блок «Что нужно исправить перед публикацией» — список всех ошибок, блокирующих публикацию.

## 9. НАВИГАЦИЯ

- «Назад» → Шаг 1.
- «Далее» → Шаг 3.
- Кнопка открытия редактора теста → навигация в редактор аттестации (или открытие в новой вкладке — [ПРЕДПОЛОЖЕНИЕ]).

---

## 10. РИСКИ И НЕЯСНОСТИ

1. **Контент подтемы — plain textarea** — нет WYSIWYG-редактора. Форматирование невозможно, что ограничивает качество учебного материала.
2. **Нет явного подтверждения удаления подтемы/темы** — `removeTopic` / `removeSection` вызываются напрямую [ПРЕДПОЛОЖЕНИЕ о confirm].
3. **Drag-and-drop на мобайле** — стрелки ↑↓ работают как fallback, но UX отличается от десктопа.
4. **Открытие редактора теста** — не ясно, происходит переход на другую страницу или открывается модальное окно. Несохранённые данные темы могут потеряться.
5. **Один открытый инлайн-редактор** — при переключении между темами форма предыдущей закрывается без сохранения. Несохранённые данные теряются молча.
6. **Нет ограничения минимального числа тем** — можно опубликовать курс с 0 тем [ПРЕДПОЛОЖЕНИЕ: проверяется валидацией публикации].

---

## 11. РЕКОМЕНДАЦИИ

1. Добавить Markdown/WYSIWYG редактор для контента подтемы.
2. Предупреждать о потере несохранённых данных при закрытии инлайн-формы или переходе.
3. Добавить явное confirm-диалог при удалении темы / подтемы.
4. Уточнить поведение кнопки «Редактировать тест» — сохранить текущие изменения перед переходом или открывать в модальном окне.
5. Добавить счётчик тем/подтем в шапке карточки шага.

5. **Нет возможности вернуться назад из подраздела к конкретной теме** — `←` всегда ведёт на `/assessments`, минуя страницу курса.

5. Отображать общее расчётное время прохождения курса в hero-summary.

- Ошибка загрузки: `errorText` + кнопка «Повторить».

## 9. НАВИГАЦИЯ

| Действие                     | Маршрут                                                   |

3. **Нет кнопки «Тест темы»** — если тема курса имеет собственный тест (assessmentId у section), нет явной точки входа в него. [ФАКТ: в `section` есть `assessmentId`, но в `TopicView` нет UI для его прохождения].

## 11. РЕКОМЕНДАЦИИ

1. Добавить прогресс-полосу по теме (X из Y подтем пройдено).
2. Добавить явную кнопку «Пройти тест темы» после завершения всех подтем, если у темы есть собственный тест.
3. Унифицировать обработку причин блокировки — всегда использовать статус-страницу для системных причин.

## 7. БИЗНЕС-ПРАВИЛА

- Если подтема содержит только материал — кнопка «Завершить материал» отмечает её пройденной.
- Если подтема содержит тест — пользователь должен пройти тест для завершения.
- Если подтема содержит и материал, и тест — сначала материал, затем тест [ФАКТ: `descriptionText` в SubtopicItem].
- `materialViewed` — флаг, что материал уже просмотрен (запрос к API при первом чтении).
- Время чтения рассчитывается из объёма контента (`calculateReadingSeconds`).

---

## 8. ВАЛИДАЦИИ И ОШИБКИ

- Таймер чтения блокирует кнопку «Завершить» до истечения минимального времени.
- Ошибка загрузки → карточка с текстом + кнопка «Повторить».
- Ошибка завершения материала → [ПРЕДПОЛОЖЕНИЕ: toast или console.error].

---

## 9. НАВИГАЦИЯ

| Действие                         | Маршрут                                                                  |
| -------------------------------- | ------------------------------------------------------------------------ |
| ← Назад                          | `/courses/:courseId/topics/:sectionId`                                   |
| После теста (пройден/не пройден) | Остаётся на странице, показывает модалку → «Продолжить» обновляет данные |
| После завершения курса           | [ПРЕДПОЛОЖЕНИЕ: редирект на статус-страницу `course_completed`]          |

---

## 10. РИСКИ И НЕЯСНОСТИ

1. **«ПИЦЦАФАБРИКА» захардкожена в коде** — это бренд конкретной компании прямо в шаблоне. При изменении бренда или мультитенантности потребуется правка кода.
2. **`v-html` с `sanitizeContent`** — контент рендерится как HTML. Необходимо убедиться, что `sanitizeContent` достаточно надёжна (защита от XSS).
3. **«⇩» и «💬» — заглушки** — та же проблема, что и на детальной странице курса.
4. **`COURSE_COMPLETION_STORAGE_KEY`** — используется localStorage для хранения контекста завершения. Не очищается явно [ПРЕДПОЛОЖЕНИЕ].
5. **Нет отображения числа оставшихся попыток** — пользователь не знает, сколько попыток у него осталось на тест подтемы.
6. **ReadingTimerNotice не описан** — неизвестно точное визуальное представление таймера.

---

## 11. РЕКОМЕНДАЦИИ

1. Вынести название бренда в конфигурацию (env или store).
2. Убрать или явно задизейблить кнопки «Скачать» и «Обсудить».
3. Добавить отображение числа доступных попыток теста.
4. Провести аудит `sanitizeContent` на предмет полноты защиты от XSS.
5. Добавить очистку `COURSE_COMPLETION_STORAGE_KEY` после использования.
6. Показывать процент прочитанного для длинных материалов.

# SCREEN: CLIENT — Страница статуса курса {#screen-client-status-page}

## 1. НАЗНАЧЕНИЕ

Информационная/системная страница, которая объясняет пользователю причину недоступности темы/подтемы или сообщает о завершении курса. Не является полноценным экраном обучения — это промежуточный экран-объяснение.

---

## 2. ЦЕЛЬ ПОЛЬЗОВАТЕЛЯ

Получить чёткое объяснение ситуации (блокировка, завершение, истечение срока) и понять, что делать дальше.

---

## 3. ОПИСАНИЕ LAYOUT

Страница использует компонент `StatusPageLayout` с параметрами:

- Иконка (символ)
- Тон цветовой схемы
- Заголовок
- Описание
- Основная кнопка (primary)
- Вторичная кнопка (secondary)

Контент занимает весь экран, центрирован.

---

## 4. UI-ЭЛЕМЕНТЫ — конфигурация по типу статуса

| Тип статуса               | Иконка | Тон     | Заголовок                    | Описание                              | Primary           | Secondary       |
| ------------------------- | ------ | ------- | ---------------------------- | ------------------------------------- | ----------------- | --------------- |
| `course_completed`        | ✓      | success | Курс завершен                | Вы прошли все обязательные шаги       | К списку курсов   | На главную      |
| `final_assessment_passed` | ★      | success | Итоговая аттестация пройдена | Итоговый тест сдан успешно            | Вернуться к курсу | К списку курсов |
| `attempts_exhausted`      | !      | warning | Попытки закончились          | Обратитесь к управляющему             | Вернуться к курсу | К списку курсов |
| `course_closed`           | ×      | danger  | Курс закрыт                  | Курс закрыт администратором           | К списку курсов   | На главную      |
| `sequence_blocked`        | ↗      | info    | Шаг недоступен               | Завершите предыдущий обязательный шаг | Вернуться к курсу | К списку курсов |
| `course_expired`          | ⌛     | warning | Срок курса истек             | Обратитесь к управляющему             | К списку курсов   | На главную      |

**Fallback**: при неизвестном `statusType` → используется `sequence_blocked`.

---

## 5. ВЗАИМОДЕЙСТВИЯ

**Primary кнопка:**

- `course_completed`, `course_closed`, `course_expired` → `/assessments`
- Остальные → `/courses/:courseId`

**Secondary кнопка:**

- `final_assessment_passed`, `attempts_exhausted`, `sequence_blocked` → `/assessments`
- Остальные → `/dashboard`

---

## 6. СОСТОЯНИЯ

Страница статична — нет loading-состояния. Конфигурация определяется из роут-параметра `statusType`.

---

## 7. БИЗНЕС-ПРАВИЛА

- `courseId = 0` при отсутствии параметра → кнопка «Вернуться к курсу» ведёт на `/assessments`.
- Неизвестный `statusType` → graceful fallback на `sequence_blocked`.

---

## 8. ВАЛИДАЦИИ И ОШИБКИ

- Нет API-запросов — страница чисто роутинговая. Нет состояний ошибки.

---

## 9. НАВИГАЦИЯ

| Кнопка    | Назначение         | Маршрут                           |
| --------- | ------------------ | --------------------------------- |
| Primary   | Зависит от статуса | `/assessments` или `/courses/:id` |
| Secondary | Зависит от статуса | `/assessments` или `/dashboard`   |

---

## 10. РИСКИ И НЕЯСНОСТИ

1. **Описание `attempts_exhausted` и `course_expired`** — «Обратитесь к управляющему» без контактов или механизма обращения. Пользователь не знает, как именно обратиться.
2. **Нет кнопки «назад»** — только системная кнопка браузера. При прямом переходе по URL пользователь теряет навигационный контекст.
3. **`course_completed` не показывает результаты** — пользователь видит «Курс завершен», но не видит итогового балла, сертификата или достижений.
4. **`final_assessment_passed` и `course_completed` — разные экраны** — возможна путаница: пользователь сдал итоговую аттестацию → `final_assessment_passed`, но не видит явного перехода к состоянию «курс завершён».
5. **Иконки — текстовые символы** (`✓`, `★`, `!`, `×`, `↗`, `⌛`) — могут отображаться по-разному в зависимости от платформы/шрифта.

---

## 11. РЕКОМЕНДАЦИИ

1. Для `attempts_exhausted` и `course_expired` добавить кнопку или ссылку «Написать управляющему» (через Telegram или чат).
2. Добавить кнопку «Назад» как третий вариант навигации.
3. На экране `course_completed` показать итоговый балл, список пройденных тем и, если реализовано, бейдж/сертификат.
4. Связать `final_assessment_passed` с автоматическим обновлением статуса курса и редиректом на `course_completed` при соответствии условиям.
5. Заменить текстовые символы на SVG-иконки для консистентности.
6. Добавить аналитическое событие на каждый тип статуса для отслеживания частоты блокировок.
