# Тест-кейсы QA для Sprint 2 (модуль «Курсы»)

Обновлено: 22.04.2026

## 1. Список курсов

### TC-CL-01: Отображение всех статусов

- Предусловия: у пользователя назначены курсы в статусах `not_started`, `in_progress`, `completed`, `closed`.
- Шаги:
1. Открыть экран «Обучение».
2. Убедиться, что по умолчанию активна вкладка «Курсы».
3. Проверить карточки курсов.
- Ожидаемо:
1. Для каждого курса отображается корректный статус.
2. Отображаются срок, количество тем, тестов и прогресс.

### TC-CL-02: Empty state

- Предусловия: у пользователя нет назначенных курсов.
- Шаги:
1. Открыть экран «Обучение».
- Ожидаемо:
1. Показано состояние «Курсы пока недоступны».

### TC-CL-03: Error state

- Предусловия: API `/courses` возвращает ошибку.
- Шаги:
1. Открыть экран «Обучение».
- Ожидаемо:
1. Экран не падает.
2. Показано пустое состояние без крэша.

## 2. Страница курса (блокировки + BottomSheet)

### TC-CD-01: Блокировка темы курса

- Предусловия: следующая обязательная тема заблокирована.
- Шаги:
1. Открыть страницу курса.
2. Нажать на `LockBadge` заблокированной темы.
- Ожидаемо:
1. Открывается `BottomSheet` с причиной блокировки.

### TC-CD-02: Завершение подтемы по времени

- Предусловия: подтема содержит материал без теста.
- Шаги:
1. Вызвать `POST /courses/:courseId/topics/:topicId/start`.
2. Попробовать завершить раньше минимального времени.
3. Повторить завершение после истечения времени.
- Ожидаемо:
1. Раннее завершение возвращает `409`.
2. После истечения времени подтема завершена и прогресс обновлён.
