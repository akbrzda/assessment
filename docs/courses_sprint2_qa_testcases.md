# Тест-кейсы QA для Sprint 2 (модуль «Курсы»)

Обновлено: 22.04.2026

## 1. Список курсов

### TC-CL-01: Отображение всех статусов

- Предусловия: у пользователя назначены курсы в статусах `not_started`, `in_progress`, `completed`, `closed`.
- Шаги:
1. Открыть экран «Обучение».
2. Убедиться, что по умолчанию активна вкладка «Курсы».
3. Проверить карточки курсов.
- Ожидаемо:
1. Для каждого курса отображается корректный статус.
2. Отображаются срок, количество тем, тестов и прогресс.

### TC-CL-02: Empty state

- Предусловия: у пользователя нет назначенных курсов.
- Шаги:
1. Открыть экран «Обучение».
- Ожидаемо:
1. Показано состояние «Курсы пока недоступны».

### TC-CL-03: Error state

- Предусловия: API `/courses` возвращает ошибку.
- Шаги:
1. Открыть экран «Обучение».
- Ожидаемо:
1. Экран не падает.
2. Показан блок ошибки с текстом и кнопкой «Повторить».
3. По нажатию «Повторить» выполняется повторная загрузка.

## 2. Страница курса (блокировки + BottomSheet)

### TC-CD-01: Блокировка темы курса

- Предусловия: следующая обязательная тема заблокирована.
- Шаги:
1. Открыть страницу курса.
2. Нажать на `LockBadge` заблокированной темы.
- Ожидаемо:
1. Открывается `BottomSheet` с причиной блокировки.

### TC-CD-02: Завершение подтемы по времени

- Предусловия: подтема содержит материал без теста.
- Шаги:
1. Вызвать `POST /courses/:courseId/topics/:topicId/start`.
2. Попробовать завершить раньше минимального времени.
3. Повторить завершение после истечения времени.
- Ожидаемо:
1. Раннее завершение возвращает `409`.
2. После истечения времени подтема завершена и прогресс обновлён.

### TC-CD-03: StickyFooterCTA — старт курса

- Предусловия: курс в статусе `not_started`.
- Шаги:
1. Открыть страницу курса.
2. Нажать нижнюю фиксированную кнопку `Начать прохождение`.
- Ожидаемо:
1. Отправляется `POST /courses/:id/start`.
2. Статус курса меняется на `in_progress`.
3. Кнопка переключается в состояние `Продолжить курс`.

### TC-CD-04: StickyFooterCTA — продолжение курса

- Предусловия: курс в статусе `in_progress`.
- Шаги:
1. Открыть страницу курса.
2. Нажать нижнюю фиксированную кнопку `Продолжить курс`.
- Ожидаемо:
1. Экран скроллится к списку тем курса.

## 3. Страница темы курса

### TC-TV-01: Успешная загрузка темы

- Предусловия: `GET /courses/:courseId/sections/:sectionId` возвращает тему с подтемами.
- Шаги:
1. На странице курса нажать `Открыть тему`.
- Ожидаемо:
1. Открывается `TopicView`.
2. Отображается список подтем и корректные статусы.

### TC-TV-02: Error state страницы темы

- Предусловия: `GET /courses/:courseId/sections/:sectionId` возвращает ошибку.
- Шаги:
1. Открыть страницу темы.
2. Нажать `Повторить`.
- Ожидаемо:
1. Показан блок ошибки без крэша.
2. По нажатию `Повторить` выполняется повторный запрос.

### TC-TV-03: BottomSheet причины блокировки подтемы

- Предусловия: в теме есть подтема со статусом `locked` или `completed_locked`.
- Шаги:
1. На странице темы нажать `Причина блокировки`.
- Ожидаемо:
1. Открывается `BottomSheet`.
2. В `BottomSheet` отображается `lockReasonText` из API.
