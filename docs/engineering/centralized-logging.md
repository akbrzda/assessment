# Централизованный сбор логов (Loki/ELK)

## Цель
Собрать backend и bot логи в одном месте для расследований и алертов, сохранив текущий `winston`-формат JSON.

## Базовый вариант (Loki)
1. Оставить запись в файл `logs/app.log`.
2. Поднять `promtail` рядом с сервисами и читать:
- `backend/logs/*.log`
- `bot/logs/*.log` (если включен файловый транспорт)
3. Отправлять в `Loki` с метками:
- `service=backend|bot`
- `env=dev|stage|prod`
- `level=info|warn|error`
4. В Grafana настроить:
- дашборды по ошибкам,
- сохраненные запросы по `correlationId`, `userId`, `route`.

## Альтернативный вариант (ELK)
- Filebeat -> Logstash -> Elasticsearch.
- Поля и метки идентичны Loki-варианту для сохранения совместимости дашбордов.

## Требования к логам
- Формат: JSON (уже реализован через `winston`).
- Нельзя логировать секреты (`token`, `password`, `jwt`).
- Для ошибок обязательно: `message`, `stack`, бизнес-контекст.

## План внедрения
1. Включить `requestId/correlationId` в middleware.
2. Прокидывать `requestId` в сервисные логи.
3. Настроить promtail pipeline для парсинга JSON.
4. Добавить 3 алерта:
- всплеск `error` > N/мин,
- рост `5xx`,
- недоступность источника логов.
