# Offline graceful degradation для MiniApp

## Цель
При потере сети MiniApp не должен выглядеть "сломавшимся": пользователь получает понятный статус, а критичные действия блокируются безопасно.

## Текущее решение (Sprint 6)
- В `frontend/src/App.vue` добавлен глобальный offline banner.
- Состояние сети отслеживается через `online/offline` события браузера.
- При offline пользователь видит, что часть функций временно недоступна.

## Поведение интерфейса
- Read-only экраны продолжают показывать уже загруженные данные.
- Операции записи (отправка ответа, сохранение формы) должны:
1. блокироваться в UI,
2. показывать понятное сообщение,
3. предлагать повтор после восстановления сети.

## Следующий шаг (после Sprint 6)
1. Добавить легковесный offline cache для последних успешных GET-запросов.
2. Для критичных POST-операций хранить локальную очередь retries.
3. Реализовать стратегию "replay after reconnect" с защитой от дублей на backend.

## Ограничения
- Без Service Worker не гарантируется полноценная offline-работа после полного перезапуска WebView.
- Offline режим не должен обходить валидацию или бизнес-правила backend.
