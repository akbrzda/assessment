# TASKS — Стабилизация и доработка существующего функционала

> Модуль «Курсы» завершён. Все задачи ниже взяты из Roadmap (приоритет 2) и отчётов аудита.
> Порядок: сначала безопасность, затем логика и удобство, затем операционная готовность.

---

## 1. БЕЗОПАСНОСТЬ — P0/P1

### 1.1 JWT-секреты — fail-fast

- [ ] Убрать fallback-значения (`your_secret_key_here`) из `backend/src/config/env.js`
- [ ] Добавить проверку при старте: если `JWT_SECRET` или `JWT_REFRESH_SECRET` не заданы — аварийно завершать процесс с понятной ошибкой
- [ ] Убрать fallback из `backend/src/middleware/verifyJWT.js` и `backend/src/controllers/adminAuthController.js`

### 1.2 CORS — строгий origin для production

- [ ] Запретить `origin: true` при пустом `ALLOWED_ORIGINS` (`backend/src/app.js`)
- [ ] Добавить валидацию: если `ALLOWED_ORIGINS` не задан в production — аварийно завершать процесс
- [ ] Убедиться, что WebSocket также получает строгий `allowedOrigins`

### 1.3 CSRF-защита для refresh cookie-flow

- [ ] Реализовать double-submit cookie или проверку `Origin`/`Referer` на маршруте `POST /admin/auth/refresh`
- [ ] Убедиться, что `SameSite` у refresh-cookie корректно выставлен для production

### 1.4 Rate-limit на admin login

- [ ] Подключить `express-rate-limit` на `POST /admin/auth/login`
- [ ] Настроить: не более 10 попыток за 15 минут с одного IP, затем блокировка на 15 минут
- [ ] Добавить информативный ответ при блокировке (429 + сообщение)

### 1.5 Access token — перевод из localStorage в память

- [ ] В `admin-frontend/src/stores/auth.js` хранить access token в реактивной переменной (не в `localStorage`)
- [ ] В `admin-frontend/src/utils/axios.js` брать токен из store, а не из `localStorage`
- [ ] Убедиться, что после refresh токен обновляется в памяти корректно

### 1.6 Telegram initData — проверка auth_date

- [ ] В `backend/src/services/telegramAuthService.js` добавить проверку `auth_date`
- [ ] Окно: если `auth_date` старше 5 минут относительно текущего времени — отклонять запрос (401)

### 1.7 Security headers (helmet + CSP)

- [ ] Подключить пакет `helmet` в `backend/src/app.js`
- [ ] Настроить базовый CSP, запрещающий `unsafe-inline` для скриптов
- [ ] Проверить, что `X-Frame-Options`, `HSTS`, `X-Content-Type-Options` выставлены
